Benefits of Not Repeating Code
I want to further discuss the benefits of not repeating codes. Repeating codes can throw the html structure into a downward spiral. Remember computers are stupid. You have to be extremely specific before you start adding in a multitude of coding. Think before you add the code in. Ask yourself, why am I adding this in here, what will the end result be?To avoid code repetition there are a few ways this can be acieved. One way to avoid this can be by using class names to style related elements. Another way can be to use the "inherit" code to assist you. The reason why code repetition happens is because if you dont feel like writing a code, and rather you would prefer to copy and paste the code in, you may be duplicating the efforts here.
Why Structured Code is so important
A structured code is so important because it gives tells your website "here is what the website is going to look like." Having an HTML opener and an HTML closer for example tells the website that this code will be written in HTML format. Using codes such as "h1" or h2 tells the website what sizings the font will be displayed in. Finally, adding Div tags in. This tells the website that there is a divide within the paragraphs. The code structure gives the website the body, and the CSS code helps to design it.
